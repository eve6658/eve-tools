## 通达信 L2 协议逆向工程

### 背景
通过 Frida 动态注入 hook AX.exe（通达信客户端）的 recv/WSARecv 函数，捕获并分析网络数据流。

### 协议格式（已确认）

```
帧结构: FD FD FD FD [8字节ASCII hex长度] [payload]
压缩:   zlib (magic: 78 9c)
数据:   JSON，GBK编码
路由:   route 字段分发消息类型
```

#### 帧头字段解析
- `FD FD FD FD` — 4字节 magic marker
- `0000001e` — 8位 ASCII 十六进制，表示 payload 长度（例：0x1e = 30字节）
- payload — 压缩或明文数据

#### Payload 子头（zlib消息）
- `[0:4]` — 消息类型/长度？
- `[4:8]` — 序号？
- `[8:12]` — 数据长度？
- `[12:]` — zlib 压缩的 JSON

| 大小 | 类型 | 说明 |
