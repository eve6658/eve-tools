================================================================================
 OpenClaw Secrets 配置教程 — 将明文 API Key 迁移到环境变量引用
================================================================================

适用场景：
  - openclaw.json 或 auth-profiles.json 中存储了明文 API Key
  - 想把 key 移到 .env 文件中，通过 SecretRef 引用，提高安全性

前置条件：
  - 已安装 OpenClaw
  - 已有 ~/.openclaw/.env 文件（没有就先创建）

================================================================================
 第一步：准备工作 — 创建/编辑 .env 文件
================================================================================

命令：nano ~/.openclaw/.env

内容示例（按需填写）：
  GATEWAY_TOKEN=你的gateway_token值
  XIAOMI_API_KEY=你的小米api_key值
  OPENROUTER_API_KEY=你的openrouter_key值
  TAVILY_API_KEY=你的tavily_key值

说明：每行一个，格式：变量名=值，不要有空格或引号
保存退出：Ctrl+O 回车，Ctrl+X

================================================================================
 第二步：启动交互式配置
================================================================================

命令：openclaw secrets configure

================================================================================
 第三步：Configure secret providers
================================================================================

选择：Continue（按回车）

解释：env 方式已可用，不需要额外添加 provider

================================================================================
 第四步：Select credential field
================================================================================

用上下箭头选择要转换的字段，按回车确认

建议处理顺序：
  1. gateway.auth.token              ← 最重要
  2. models.providers.xiaomi.apiKey  ← 小米模型 key
  3. profiles.xiaomi:default.key     ← 小米认证 profile
  4. profiles.openrouter:default.key ← OpenRouter key
  5. skills.entries.tavily-search.apiKey

注意：
  key = API 密钥（大多数 provider 用这个）
  token = Bearer Token（一般不用改）

================================================================================
 第五步：Secret source
================================================================================

选择：直接按回车确认（选 env）

================================================================================
 第六步：Provider alias
================================================================================

选择：直接按回车，保持默认 "default"

================================================================================
 第七步：选择环境变量
================================================================================

选对应的环境变量名，按回车确认
如处理 xiaomi key 就选 XIAOMI_API_KEY

================================================================================
 第八步：Configure another credential?
================================================================================

还有要处理的 → Yes
只改一个 → No

================================================================================
 第九步：确认并执行 Plan
================================================================================

检查变更预览，确认无误后选 Apply 执行

================================================================================
 第十步：验证结果
================================================================================

命令：
  openclaw secrets audit --check
  openclaw secrets reload

期望：unresolved=0，plaintext 数量减少
.env 中的 PLAINTEXT_FOUND 是正常的

================================================================================
 常见问题
================================================================================

Q: 改错了怎么办？
A: 重新运行 openclaw secrets configure 重新映射即可

Q: models.json 里的明文 key 需要手动改吗？
A: 不需要，自动生成的

Q: env allowlist 怎么填？
A: 填 .env 中的变量名，逗号分隔，如：XIAOMI_API_KEY,TAVILY_API_KEY

 快速参考
================================================================================

  openclaw secrets configure
